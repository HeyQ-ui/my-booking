# 开发文档 · 项目概况与进程

面向开发者的项目内部说明：技术选型、目录结构、约定、已知问题与开发进程。

## 一、项目概况

- **定位**：中山大学体育场馆预约流程的纯前端演示 / 原型。
- **技术栈**：原生 HTML + CSS + JavaScript，无框架、无构建、无依赖、无后端。
- **运行方式**：浏览器直接打开 HTML；无 npm、无 lint、无自动化测试，验证靠手动走查流程。
- **状态管理**：页面间用 URL 查询参数传递，无持久化存储。

### 三步流程

```
index.html            booking.html                   我的预约.html
  选日期      ──date──►   选场次（硬编码数据）  ──params──►  预约结果 + 详情悬浮层
```

参数链：`date` → `date` → `date, venue, time, type`。

## 三、关键实现说明

### 1. `index.html` / `booking.html`（手写页）
常规静态页。`booking.html` 的场次数据硬编码在 `sessions` 数组，`campusGroups` 定义校区→场馆分组，`renderSessions()` 渲染折叠栏并支持搜索。日期一律用本地时区手拼 `YYYY-MM-DD`（`todayLocalStr()`），避免 `valueAsDate` 的 UTC 偏移。

### 2. `我的预约.html`（改造型页面）—— 核心难点
该文件是用浏览器 **SingleFile** 插件保存的中山大学官方 Vuetify 2.7 页面（约 1MB，含内联 CSS / base64 资源 / 压缩单行）。我们的定制代码只追加在文件**末尾**：

- **`<style>` 块**：作用域前缀 `.booking-detail-*`，补齐官方页缺失的 Vuetify 规则（`.v-dialog`、`.v-dialog__content`、`.v-overlay` 遮罩、`.v-card` 白底、主按钮白字等）。
- **第一段 `<script>`**：读取 URL 参数，改写官方预约卡片的标题 / 日期 / 时段 / 场地；处理「返回」按钮；时段统一格式为 `18:00 - 20:00`。
- **第二段 `<script>`**：点击 `.card-item` 弹出详情悬浮层。悬浮层 HTML 直接复刻自 `我的预约（点击1）.html`，内容由 `parseCard()` 从被点卡片实时解析，保证与卡片一致；确认按钮 / 遮罩点击关闭；初始化时移除卡片「取消」按钮。

> 复刻悬浮层的教训：官方页面之间 CSS 规则并不完整对齐——`我的预约.html` 缺了 `我的预约（点击1）.html` 里的若干 `.v-dialog`/`.v-card`/按钮白字规则。只拷 HTML 不拷 CSS 会导致悬浮层错位、字色/字重不符。

## 四、开发约定

- **Commit message 用中文**（沿用历史风格）。
- 修改超大 HTML 文件时：`grep -oP` 定位片段 + `Edit` 精确替换，勿整文件重写；正则要有界，避免在超长行上灾难性回溯。
- 悬浮层相关改动，以 `我的预约（点击1）.html` 为视觉/结构「标准答案」逐项比对。
- 保持官方 DOM class 选择器不变（`.card-item`、`.booking-title`、`.card-details`、`.v-dialog` 等）。
